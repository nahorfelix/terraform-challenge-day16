4) LIFECYCLE
prevent_destroy on state bucket + lock table: stops accidental destroy of remote state — without it, deleted state = lost infra map and possible prolonged outage.
create_before_destroy on LT/SG/ASG: new resource before delete — without in-place updates, destroys first can cause downtime or dependency errors.

5) CLOUDWATCH ALARM (paste)
resource "aws_cloudwatch_metric_alarm" "high_cpu" {
  count = local.monitoring_enabled ? 1 : 0
  metric_name = "CPUUtilization"
  namespace = "AWS/EC2"
  period = 120
  evaluation_periods = 2
  threshold = var.cpu_alarm_threshold
  dimensions = { AutoScalingGroupName = aws_autoscaling_group.web.name }
  alarm_actions = concat([aws_sns_topic.alerts.arn], ...)
}
80% default (~4 min sustained). Fires: SNS + optional scale-out.
